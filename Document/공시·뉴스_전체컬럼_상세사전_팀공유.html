1.tb_disclosure — 공시 19컬럼 전체

저장 순서 그대로. 공시는 “1건이 곧 사실”이라 집계가 없고, 원문 식별(제목·번호·시각)이 고유합니다.

1tickerstr🕓종목 티커. 유니버스(분석 대상 목록)에서 들어온 대문자 심볼.✗AAPL

4has_signal0/1🔗신호 유무. LLM/코드 신호가 있으면 1, 없으면 0.✓1

15confidence0~1🤖→⚙️LLM 확신도. certainty_level(High .9/Med .6/Low .3)을 조회.✗0.9
16hard_block0/1⚙️즉시 매수 차단. 최종 권한이 BLOCK_ALL/BLOCK_BUY(_BLOCK)면 1.✓0
17hard_block_reasonstr|None⚙️차단 사유(권한 정책의 final_reason). 전달 시 summary 앞에 [차단:…] 삽입.✗ →summarynull
18summarystr🤖공시 전문 3~4줄 요약(영어). 차단 시 앞에 사유 접두.✓"Microsoft agreed to acquire…"
19keywordsstr(list)🤖핵심 키워드 5개. 저장 시 ", "로 접어 TEXT로.✗"acquisition, cash, cybersecurity"

2.tb_news — 뉴스 26컬럼 전체

저장 순서 그대로. 뉴스는 여러 기사를 신뢰가중 집계하므로 importance/risk/source_trust/grade_score 등이 “가중 평균” 산출입니다. 방향·요약·원문식별은 대표 기사(신뢰가중 최고) 값을 씁니다.

3.세 분류 — 공통 / 공시전용 / 뉴스전용

두 테이블을 겹쳐 보면 이렇게 나뉩니다. 대부분은 공통이고, 차이는 “집계 유무”와 “원문 식별 방식”에서 옵니다.

분류컬럼왜
공통(양쪽)ticker · collected_at · trade_date · has_signal · event_type · sentiment · sentiment_score · is_positive · reason · importance · risk_score · confidence · hard_block · hard_block_reason · summary · keywords방향·강도·위험·안전장치·요약은 두 소스가 같은 축을 공유.
공시 전용filing_title · filing_no · filed_at공시 원문 식별(제목·번호·업로드시각).
뉴스 전용news_title · source · published_at · ref · news_count · disclosure_ref · peak_importance · source_trust · grade_score · top_evidence여러 기사 집계(건수·peak·매체신뢰·등급·근거) + 기사 원문 식별 + 공시 교차참조.
기억법. 공시에 없는 뉴스 컬럼은 전부 “여러 건을 다루기 때문에” 생긴 것(집계·건수·매체신뢰)이거나 “기사 출처가 다양하기 때문에” 생긴 것(source·grade·ref)입니다. 공시는 1건·1출처(SEC)라 이것들이 필요 없어요.

4.계산 컬럼 공식 요약

위 표에서 ⚙️로 표시된 계산 컬럼들의 근거 공식입니다(상세·예시는 두 완전 가이드 §공식).

① _n10 (0~10 → 0~1)
결과 = (값을 0~10로 자름) ÷ 10, round(2)          # 7 → 0.7

② _news_conf (뉴스 기사별 무게)
무게 = 확신도 × 확인계수 × 매체신뢰
       확신도 High.9/Med.6/Low.3 · 확인계수 확정1.0/소문0.5

③ _senti_score (방향 0~1, 호재만 감쇠)
방향강도 = 부호 × 강도;  if 방향강도>0: ×= 확신도
sentiment_score = 0.5 + 0.5 × 방향강도            # 악재는 감쇠 안 함

④ 신뢰가중 평균 (뉴스 importance·risk·source_trust·grade_score)
지표 = Σ(값 × 무게) ÷ Σ(무게)                      # 무게 전부 0이면 등가중 폴백

⑤ peak_importance (뉴스)
peak = max( importance, 신뢰기사들의 강도… )       # 신뢰기사 = 확정/trust≥.6/ALLOW

※ 공시는 집계가 없어 ②④⑤ 미사용. confidence = certainty_level 직접 조회.

5.전달 계약 매핑(개명) 한눈에

저장 컬럼 중 전략가에게 나가는 것만, 개명을 포함해 정리합니다.

공시 전달 7필드 = has_signal · event_type · importance_score(←importance) · sentiment_score · risk_score · hard_block · summary
뉴스 전달 10필드 = has_signal · article_count(←news_count) · event_type · importance_score(←importance) · peak_importance · sentiment_score · risk_score · trust_score(←source_trust) · hard_block · summary
저장 컬럼전달 이름비고
importanceimportance_score양쪽 공통 개명(_score 접미로 “0~1 점수” 명시)
source_trusttrust_score뉴스만
news_countarticle_count뉴스만(의미 명확화)
hard_block_reason(별도 필드 없음)차단 시 summary 앞 [차단:…]로 흡수
주의. 개명은 전달 계약에서만 적용됩니다. 저장 컬럼명(importance·source_trust·news_count)과 내부 LLM 스키마는 그대로예요(파급 최소화). 그래서 이 사전(저장 기준)과 전달 JSON의 키 이름이 일부 다릅니다.
