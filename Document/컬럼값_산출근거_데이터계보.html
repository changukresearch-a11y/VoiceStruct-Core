§ 범례 — 원천(Source) 5종

각 컬럼 카드 우상단 칩이 그 값의 1차 원천이다. 대부분은 LLM 또는 SEC/RSS 원시값을 CODE가 정규화·집계해 결정한다.

§ 공통 산출 헬퍼 — 모든 컬럼이 참조

컬럼 계보에서 반복 인용되는 정규화·가중 함수. 한 곳에 정의하고 각 컬럼에서 이름으로 참조한다.

_n10 — 0~10 척도를 0~1로

def _n10(v): # LLM은 0~10, DB는 0~1로 통일 return round(max(0.0, min(10.0, float(v or 0))) / 10.0, 2) # importance 7 → 0.7 · risk_score 3 → 0.3 · None → 0.0

_CERTAINTY — LLM 확신도 라벨을 confidence(0~1)로

_CERTAINTY = {"High": 0.9, "Medium": 0.6, "Low": 0.3} # 미지정 기본 0.5

_sign — 방향 라벨을 부호로

_sign(sentiment) = {"positive": +1, "negative": −1}.get(sentiment, 0) # neutral·mixed = 0

_senti_score — 방향×강도를 0~1로 (호재만 신뢰 감쇠)

def _senti_score(signed, confidence=1.0): signed = clip(signed, −1, +1) if signed > 0: # 호재(+)만 confidence로 중립(0.5)쪽 감쇠 signed *= clip(confidence, 0, 1) # 근거 약한 낙관 과신·펌프 방지 # 악재(−)는 감쇠 안 함 — 안전 우선, 경고 신호 안 흐림('가장 보수적 채택') return round(0.5 + 0.5 * signed, 2) # 0=강악재·0.5=중립·1=강호재

_news_conf — 기사별 신뢰가중치 weight (뉴스)

def _news_conf(sig): c = _CERTAINTY.get(sig.certainty_level, 0.5) # 확신도 c *= 1.0 if sig.is_confirmed else 0.5 # 확정=1.0 / 추측=0.5 c *= sig.source_trust if source_trust is not None else 1.0 # 0.0을 1.0로 오인 금지 return clip(c, 0, 1)

_news_credible — peak_importance 산정 자격

def _news_credible(r): # 신뢰검증된 기사인가 return r.signal.is_confirmed or r.signal.source_trust ≥ 0.6 or r.source_grade == "ALLOW"

_GRADE_SCORE — 출처 정책등급을 점수로 (코드·도메인·사전)

_GRADE_SCORE = {"ALLOW": 1.0, "GRAY": 0.6, "BLOCK": 0.0} # 미등급 기본 0.6(GRAY)

weight · tw — 뉴스 집계 가중치

raw_weights = [_news_conf(r.signal) for r in analyzed] weights = raw_weights if sum(raw_weights) > 0 else [1.0]*n # 전부 0이면 등가중 폴백 tw = sum(weights) # 모든 신뢰가중 평균의 분모
🔑 대표 기사(rep) 선정. 뉴스 B그룹(제목·출처·event_type 등)은 scored = sort(analyzed, key = importance×weight desc)의 1위 기사에서 복사한다. 시끄럽지만 저신뢰인 루머가 대표가 되지 않도록 강도×신뢰로 정렬(§신뢰가중 심화 참조).

결정
평균(15번)에 희석되기 전 가장 센 믿을 만한 한 방. 신뢰 기사 없으면 importance로 수렴(peak ≥ importance 불변식). peak>importance면 low-conf 딱지. 저신뢰 과장이 가짜 peak 세우는 편향 차단.
예 확정 대형기사 imp0.9 + 루머 imp0.3 → importance(가중평균)=0.5여도 peak=0.9. 반대로 imp0.9가 저신뢰 루머면 cred 아님 → peak=importance로 수렴.
17risk_scoreNUMERIC 0~1CODE 집계
원천
각 기사 _n10(signal.risk_score).
처리
Σ(_n10(risk) × w) / tw 신뢰가중 평균.
결정
오늘 뉴스 전체 위험도. 방향과 별개 축.
18sentiment_scoreNUMERIC 0~1CODE 집계
원천
signed = [_n10(imp) × _sign(sentiment)](방향×강도) + confidence(22번).
처리
strong_neg = min(signed) ≤ −0.7. agg = min(signed) if strong_neg else Σ(s×w)/tw → _senti_score(agg, confidence).
결정
강한 악재 1건이면 그쪽으로(보수). 아니면 신뢰가중 평균. 호재는 confidence로 감쇠, 악재 유지.
예 강악재(signed −0.8) 1건 섞이면 → agg=−0.8 → 0.1. 전부 약호재면 가중평균 후 감쇠.

결정
라벨을 최종 점수에서 파생 → 라벨↔점수 항상 일치. 호·악재 둘 다 유의미하면 mixed 신설.

결정

★ 신뢰가중치 weight — 뉴스 집계의 심장

뉴스 C그룹(15~22)과 대표선정·top_evidence가 모두 이 기사별 weight에 의존한다. weight를 한 번 이해하면 tb_news 집계 전체가 풀린다.
# 기사 1건의 신뢰가중치 (0~1) weight = certainty_factor × confirmed_factor × source_trust certainty_factor = {High:0.9, Medium:0.6, Low:0.3, 기타:0.5}[certainty_level] # LLM 확신 confirmed_factor = 1.0 if is_confirmed else 0.5 # 확정/추측 source_trust = LLM 출처신뢰 (None이면 1.0) # 0.0 오인 방지 # 집계 분모 · 폴백 weights = raw_weights (전부 0이면 [1.0]*n 등가중 — 강도/위험 0붕괴 방지) tw = Σ weights # 이 weight가 결정하는 것 importance = Σ(_n10(imp)·w)/tw risk_score = Σ(_n10(risk)·w)/tw source_trust = Σ(trust·w)/tw grade_score = Σ(_GRADE_SCORE·w)/tw sentiment_score= _senti_score(Σ(signed·w)/tw, confidence) # 강악재는 min 우선 대표기사 rep = argmax(importance × weight) → news_title·source·event_type·reason·ref top_evidence = scored[:3] confidence = mean(raw_weights)
