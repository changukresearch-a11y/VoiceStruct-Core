0.공통 예시 데이터 & 기초 상수

이 문서 전체가 아래 같은 3기사를 재료로 씁니다(엔비디아에 들어온 뉴스라고 가정).

고정 상수(코드):

_CERTAINTY  = {High: 0.9, Medium: 0.6, Low: 0.3}     # 확신도 등급→점수
_GRADE_SCORE = {ALLOW: 1.0, GRAY: 0.6, BLOCK: 0.0}    # 출처 등급→점수

1.기초 함수 — 모든 공식의 부품

1-1. _n10 — 0~10을 0~1로 기초

결과 = (값을 0~10 사이로 자름) ÷ 10, round(2)
# 원 코드: round(clamp(v,0,10)/10, 2)
7 → 0.7 · 5 → 0.5 · 9 → 0.9 · 13 → 1.0(잘림) · -2 → 0.0(잘림)

LLM은 0~10으로 답하고, 우리는 모든 점수를 0~1로 통일합니다. importance·risk가 이걸 통과해요.

1-2. _news_conf — 기사 한 건의 무게(가중치) 기초

모든 집계에서 “믿을수록 크게”의 무게로 쓰입니다. 세 요소의 곱이에요.

무게 w = 확신도 × 확인계수 × 매체신뢰
  확신도   High 0.9 / Medium 0.6 / Low 0.3   (certainty_level)
  확인계수 확정이면 1.0, 소문이면 0.5           (is_confirmed)
  매체신뢰 LLM이 본 0~1 값                     (source_trust)
A1 = 0.9 × 1.0 × 0.9 = 0.81 A2 = 0.6 × 1.0 × 0.8 = 0.48 A3 = 0.3 × 0.5 × 0.3 = 0.045 ← 소문·저신뢰라 무게 급감 tw(총 무게) = 0.81 + 0.48 + 0.045 = 1.335
코드 함정. 매체신뢰가 0.0(무신뢰)일 때 x or 1.0을 쓰면 0.0을 1.0으로 오인합니다. 그래서 st if st is not None else 1.0으로 None만 1.0 취급하고 0.0은 그대로 곱합니다.

1-3. _senti_score — 방향(-1~+1)을 0~1로 기초

방향강도 = clamp(입력, -1, +1)
if 방향강도 > 0:  방향강도 ×= confidence      # 호재만 감쇠
결과 = 0.5 + 0.5 × 방향강도                    # 악재는 감쇠 없음
눈금: -1 → 0.0(극악재) · 0 → 0.5(중립) · +1 → 1.0(극호재) 악재 예: 방향강도 -0.8 → 0.5 + 0.5×(-0.8) = 0.1 (감쇠 없이 또렷)
비대칭 감쇠. 호재만 confidence로 눌러 근거 약한 낙관을 덜 단정하고, 악재(경고)는 흐리지 않습니다 — “가장 보수적 채택”.

2.confidence — 무게의 단순평균 (÷n)

confidence = Σ(무게) ÷ 기사 수(n)      # raw_weights 사용(폴백 이전)
대상기사별 무게 _news_conf 그 자체
분모n(기사 수) — tw 아님!
confidence = (0.81 + 0.48 + 0.045) ÷ 3 = 1.335 ÷ 3 = 0.45
왜 단순평균인가. 저신뢰 기사(A3)가 한 표를 온전히 행사해 confidence를 끌어내리게 하려고요. 가중평균이면 A3가 무시돼 “근거 얇음”이 사라집니다. importance(가중)와 정반대 의도예요.
raw_weights 사용. 다른 지표는 “무게 전부 0이면 등가중 폴백”하지만, confidence는 폴백 이전 원본으로 계산 → 전부 무신뢰면 정직하게 0 근처로 낮게 나옵니다.

되먹임: confidence는 §5의 _senti_score 호재 감쇠에 다시 들어가 “약한 근거의 낙관”을 억제합니다.

3.importance — 강도의 가중평균 Σ(값·w)/tw

importance = Σ( _n10(강도) × 무게 ) ÷ tw        # 방향(부호) 없음
mags = [0.7, 0.5, 0.9] (부호 없는 강도) importance = (0.7×0.81 + 0.5×0.48 + 0.9×0.045) ÷ 1.335 = (0.567 + 0.24 + 0.0405) ÷ 1.335 = 0.8475 ÷ 1.335 = 0.63
핵심 관찰. A3는 강도 0.9로 제일 높지만 무게 0.045라 기여가 0.0405뿐 — 거의 무시됩니다. 그냥 산술평균이면 (0.7+0.5+0.9)/3 = 0.70인데, 신뢰가중은 0.63으로 정직하게 낮아요.
왜 부호(방향)를 안 넣나. 방향을 곱해 평균내면 큰 호재+큰 악재가 상쇄돼 0으로 붕괴합니다. importance는 순수 “강도” 축이라 부호를 뺍니다(방향은 §5 담당).

4.peak_importance — 신뢰기사 중 최댓값 max

def _news_credible(기사):      # 셋 중 하나면 자격
    return 확정 or 매체신뢰≥0.6 or 등급==ALLOW

cred_mags = [ 강도 for 기사 if _news_credible(기사) ]
peak_importance = max( importance, *cred_mags )
기사강도신뢰검증?peak 후보?
A10.7✅ 확정·ALLOW포함
A20.5✅ 확정·trust0.8포함
A30.9❌ 미확정·trust0.3·GRAY제외
cred_mags = [0.7, 0.5] # A3의 0.9 제외 peak = max(0.63, 0.7, 0.5) = 0.70
왜 신뢰기사만. 저신뢰 블로그가 강도만 9로 부풀린 기사가 가짜 peak를 세우는 걸 막습니다(2026-07-07 편향 수정 — 이전엔 max(mags)라 A3가 peak 0.9를 세웠음).
importance를 max에 넣는 이유. ① peak ≥ importance 항상 성립 ② 신뢰기사가 없으면 peak = importance로 수렴(0 붕괴 방지). peak = importance면 “평균 이상 튀는 신뢰신호 없음”을 뜻합니다.

5.sentiment_score — 방향 가중평균 + 매핑 (2층)

# 1층: 방향 집계
signed = _n10(강도) × 부호        # 부호: 호재+1/악재-1/중립0, 범위 -1~+1
if min(signed) ≤ -0.7:          # 강한 악재 우선(예외)
    agg = min(signed)
else:
    agg = Σ(signed × 무게) ÷ tw    # 정상: 방향 가중평균
# 2층: 0~1 매핑 (호재 감쇠)
sentiment_score = _senti_score(agg, confidence)
signed = [+0.7, +0.5, +0.9] (전부 호재) min(signed)=+0.5 > -0.7 → 강악재 예외 미발동, 가중평균 사용 agg = (0.7×0.81 + 0.5×0.48 + 0.9×0.045) ÷ 1.335 = 0.635 _senti_score(0.635, conf 0.45): 호재라 ×0.45 → 0.286 → 0.5 + 0.5×0.286 = 0.64
강한 악재 우선. 가장 나쁜 기사가 -0.7 이하로 아주 나쁘면 평균 대신 그 최악값을 채택 — 치명적 경고가 호재에 물타기되지 않게(보수 장치).
importance와 다른 점. 이 예제에선 agg(0.635)≈importance(0.63)로 우연히 비슷하지만(전부 호재라 signed=mags), 개념은 다름: importance는 부호 없는 mags를, sentiment는 부호 있는 signed를 쓰고 매핑·감쇠까지 합니다.

파생: sentiment 라벨 & is_positive

is_positive = sentiment_score > 0.5
sentiment = 강악재 → negative
          | 호재무게·악재무게 각 30%↑ → mixed
          | >.5 positive / <.5 negative / =.5 neutral
0.64 > 0.5 → is_positive = 1, 라벨 = positive (호재무게만 있어 mixed 아님)

라벨을 최종 점수에서 파생하므로 “점수 0.4인데 라벨 positive” 같은 모순이 없습니다.

6.risk_score — 위험의 가중평균 Σ(값·w)/tw

risk_score = Σ( _n10(위험) × 무게 ) ÷ tw      # 부호·감쇠 없음
위험 = [1, 0, 0] → _n10 → [0.1, 0, 0] risk_score = (0.1×0.81 + 0×0.48 + 0×0.045) ÷ 1.335 = 0.081 ÷ 1.335 = 0.06
독립된 하방 축. sentiment(방향)와 별개로 “얼마나 위험한가”만 잽니다. 호재인데 위험도 높을 수 있어요(예: 인수 발표지만 규제 불확실). importance와 같은 순수 크기 축(부호·감쇠 없음).
risk_score ≠ hard_block. risk_score는 0~1 참고 강도, hard_block은 권한 BLOCK 시 켜지는 0/1 즉시 차단. 별개 컬럼입니다.

7.source_trust — 매체신뢰의 가중평균 Σ(값·w)/tw

source_trust = Σ( 기사별 매체신뢰(LLM) × 무게 ) ÷ tw
trust = [0.9, 0.8, 0.3] source_trust = (0.9×0.81 + 0.8×0.48 + 0.3×0.045) ÷ 1.335 = (0.729 + 0.384 + 0.0135) ÷ 1.335 = 1.1265 ÷ 1.335 = 0.84
자기참조 구조. 무게 w 안에 이미 source_trust가 들어있어, 믿을 만한 기사가 “전체 신뢰”를 정하는 데 더 큰 목소리를 냅니다. 신뢰 출처 평가로 수렴 → 루머 매체가 우겨도 안 먹혀요.

여기선 (trust or 0)이 정상 — 값을 평균내는 자리라 None·0.0 모두 0 기여가 맞습니다(무게 계산과 반대).

8.grade_score — 도메인 등급의 가중평균 Σ(값·w)/tw

grade_score = Σ( 등급점수 × 무게 ) ÷ tw
  등급점수 = _GRADE_SCORE.get(source_grade, 0.6)   # 미등록=GRAY(0.6)
  ALLOW=1.0 · GRAY=0.6 · BLOCK=0.0                  # 이산 3값
등급 = [ALLOW→1.0, ALLOW→1.0, GRAY→0.6] grade_score = (1.0×0.81 + 1.0×0.48 + 0.6×0.045) ÷ 1.335 = (0.81 + 0.48 + 0.027) ÷ 1.335 = 1.317 ÷ 1.335 = 0.99
왜 source_trust(0.84)와 다른가. grade_score는 코드가 도메인만 보고 3단계로 매긴 이산값(ALLOW는 무조건 1.0), source_trust는 LLM이 내용 읽고 매긴 연속값(로이터라도 0.9). 규칙 vs LLM을 독립으로 두어 이중 교차검증합니다.

BLOCK(0.0)은 사전필터에서 이미 걸러져 집계엔 거의 안 들어옵니다(방어적 정의). grade_score는 전략가에 전달 안 됨(감사용).
