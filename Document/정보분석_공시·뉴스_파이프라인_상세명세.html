00 개요 — 파이프라인 위치 · 코드 vs LLM 분담

📥 입력: 스크리닝 50 티커📤 출력: tb_disclosure · tb_news🔁 공시 1h · 뉴스 5m담당: 정창욱

정보분석 파트는 스크리닝이 고른 50종목에 대해 공시·뉴스를 수집·분석해 구조화 신호 + 보수적 1차 매매권한 제안까지 낸다. 실제 매수권한·시장반응 검증은 07 Strategist가 맡는 별개 영역이다(범위 밖).

설계 3원칙

  • 코드=사실 / LLM=해석. 수치·형식·등급은 코드가 확정하고(XBRL·Item맵·도메인등급·Form4 파싱), LLM은 "이 사건이 호재냐 악재냐·얼마나 큰일이냐"의 해석만 맡는다.
  • 가장 보수적 채택. 하드룰·LLM·Item기본 여러 권한 후보 중 가장 제한적인 것을 최종 권한으로 삼는다. 시장반응 확인 전엔 매수계열을 WATCH_ONLY로 강등한다.
  • 모든 점수 0~1로 통일. LLM에는 이해가 쉬운 0~10으로 물어보고, 코드가 _n10()으로 0~1 정규화해 저장한다. 방향은 sentiment_score(0~1)로, 강도는 importance(0~1)로 축을 분리한다.

코드 vs LLM 처리 비중 (Form/소스별)

10-Q / 10-K코드 XBRL 수치+YoY코드 본문 하드룰LLM 수치 의미 해석
Form 4코드 XML 파싱코드 100% 규칙 스코어링— (LLM 미사용)

🛠 스택 — Python · pydantic · pydantic-ai(구조화 출력 강제). LLM 코드 기본값 anthropic:claude-haiku-4-5, 실제 운영은 .env의 LLM_ANALYST_MODEL=openai:gpt-4o-mini로 override. SEC·Google News·Yahoo는 전부 키 불필요(User-Agent만). DB=SQLite(data/quantinue.sqlite).

01 입력 계약 — 스크리닝 → 정보분석

📥 스크리닝 에이전트📤 tb_universe / companies담당: 스크리닝(주황)

스크리닝(스케일러)이 분석 대상 종목 리스트를 넘긴다. 정보분석은 종목 리스트를 입력으로만 받고(선별은 범위 밖), 점수·센티먼트류는 우리가 산출하므로 입력에 넣지 않는다.

필드필수설명
ticker✅ 필수종목 심볼(대문자). 유일한 필수값
rank권장처리 우선순위 → companies.priority로 반영
category / sector선택우리가 못 만드는 값 → 번들에 통과(pass-through)
company_name / cik선택없으면 SEC company_tickers.json에서 자동 매핑
🔌 로더 run_screening_input.py — JSON 배열(봉투 {schema,as_of,count,companies:[...]} 또는 순수배열) 적재. upsert_company가 ON CONFLICT 시 priority를 갱신 안 하므로, 로더가 priority=rank·active를 명시 UPDATE(시총순위 덮어쓰기 방지).

04 하드리스크 스캔 — LLM보다 우선하는 안전장치

🛡️ check_hard_risk(text)config/hard_risk_policy.yaml

'수익 기회 찾기'가 아니라 '절대 진입하면 안 되는 구간 막기'가 목적. 본문 키워드를 검사해 첫 매치를 HardRiskHit으로 반환하고, LLM 결과와 무관하게(또는 덮어써서) 권한을 강제한다.

HardRiskHit = { risk_type, trade_permission, risk_score_min, matched_keyword } check_hard_risk(text): # text.lower()에서 첫 매치 파산·상폐·거래정지·회계부실(going concern·material weakness)·대규모 희석 … → 매치 시 그 룰의 trade_permission(BLOCK_BUY 등) + risk_score_min 강제 → 미매치 시 None

8-K는 item.body를, 10-Q/K는 fetch_report_text로 받은 전체 본문을 스캔한다(XBRL 수치 요약엔 going concern 문구가 없으므로). LLM 분석 시 하드리스크가 있으면 signal.risk_score = max(risk_score, risk_score_min)로 보정하고 hard_risk_flag=True 표시.

05 공시 LLM 분석 — DisclosureSignal (0~10 원본)

07 tb_disclosure — ERD 18필드 (컬럼별 산출)

📤 DisclosureBundle → tb_disclosurePK=(ticker, collected_at)매 사이클 50행 append

매 공시 사이클(1h) 50종목 각각을 한 행으로 집계해 append(덮어쓰기 X, 시계열 누적). 공시 없는 종목도 has_signal=0 빈 행 → 항상 50행/사이클. Strategist는 종목별 최신 행을 읽는다. 점수는 전부 0~1.

12 집계 산출식 상세 — 신뢰가중 · 방향점수 · peak · disclosure_ref

① 기사별 신뢰가중 _news_conf

weight = certainty_factor × confirmed_factor × source_trust certainty_factor = {High:0.9, Medium:0.6, Low:0.3}[certainty_level] confirmed_factor = 1.0 if is_confirmed else 0.5 source_trust = sig.source_trust (None이면 1.0 — '0.0을 1.0으로 오인' 버그 방지) → clip(0,1). 전부 0이면 가중평균만 등가중 폴백(강도·위험 0붕괴 방지), confidence는 raw 평균이라 정직하게 낮게 유지.

② 방향점수 _senti_score(signed, confidence) — 비대칭 신뢰 감쇠

signed = clip(sign(sentiment) × importance, -1, +1) if signed > 0: signed ×= confidence ← 호재만 신뢰로 중립(0.5)쪽 감쇠 # 악재(−)는 감쇠 안 함 — 안전 우선, 경고 신호 안 흐림('가장 보수적 채택') return round(0.5 + 0.5 × signed, 2) # 0=강악재·0.5=중립·1=강호재

뉴스 방향 집계: 강한 악재(min(signed)≤−0.7) 1건이면 최솟값 채택(보수), 아니면 신뢰가중 평균. 라벨은 최종 sentiment_score에서 파생 — 강악재→negative, 호·악재 각 신뢰가중≥30%→mixed, 그 외 >.5=positive/<.5=negative/=.5=neutral.

③ 대표 강도 peak_importance — 신뢰검증 최댓값

_news_credible(r) = is_confirmed OR source_trust≥0.6 OR source_grade==ALLOW cred_mags = [_n10(imp) for r in analyzed if _news_credible(r)] peak_importance = round(max([importance] + cred_mags), 2) # peak ≥ importance 불변식

저신뢰 매체의 자극적 과장이 가짜 peak를 세우는 편향 차단. 신뢰 기사가 없으면 importance로 수렴. peak>importance일 때만 low-conf 딱지. 🔴 명세 협의 명세 최최종이 본문에 이 코드 정의(신뢰검증 강도 최댓값)를 채택 — 유지.

④ disclosure_ref 매칭 disclosure_ref_for(ticker, event_type) 🆕

if event_type in (None, "other"): return None # 광범위 오매칭 방지 후보 = 오늘 같은 종목 공시 중 norm_event(sig.event_type)==event_type 이고 accession_no 있는 것 return 후보 중 importance 최대 1건의 filing_no (없으면 None)

설계: accumulator(오늘 프로세스 메모리) 기반 · event_type 일치만 · tie는 importance 최대. build_news_bundle은 순수 함수로 유지하고 매칭은 스케줄러에서 주입(has_signal일 때만). 한계: 재시작 직후엔 오늘 공시 누적이 비어 매칭이 잠깐 빌 수 있음 → 다음 공시 틱 이후 최신 행에 채워짐.

⑤ grade_score 등급 점수

_GRADE_SCORE = {ALLOW:1.0, GRAY:0.6, BLOCK:0.0} # 코드·도메인·사전 grade_score = Σ(_GRADE_SCORE[source_grade] · w) / tw # 신뢰가중 평균
